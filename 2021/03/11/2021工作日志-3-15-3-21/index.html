2021工作日志(3.15-3.21)

发表于 2021-03-11 分类于 工作日志 阅读次数：
本文字数： 5.6k 阅读时长 ≈ 5 分钟

(注1：今天是2021年3月11日，下周一就要去新公司上班了，也不知道自己可以挺多久，自己目前的水平能做到什么程度。昨天去新公司看了一下，技术负责人给我简单介绍了一下他们公司所做的一些项目，大体是一些WebGL和Three.js做的一些三维可视化展示(类似于谷歌地图的，可以跟着箭头移动，里面的图片什么的都可以点击)，还有一些学生后台管理系统，里面的功能我看了是真的复杂，和我之前做的一个Vue电商管理系统后台完全都不能比。数据库是那个技术负责人自己搭建的，然后后台他是在网上找了一个模版，然后自己再往上面添加了一些功能就成了，他还说他的Vue是只用了一个月就学完了，说非常简单，没什么东西，然后就快速上手了。用了一个月时间，前后端都是他就完成了一个项目。听他说完我的内心有一种深深的无力和挫败感，感觉自己现在的所学简直就是渣渣，自己能否能适应他们公司的节奏，实际给我安排什么事情，一切都是未知数。)

(注2：昨天心态炸裂，有点崩溃，然后去超市买了很多吃的，搞得现在体重又上升了，随后又下载了王者荣耀玩了一晚上，发泄自己内心的压力。)

(注3：不过自己千万不要自乱阵脚，首先，核心原则一定要把握住：对我一个自学转行的人来说，前期的进步空间是极其巨大的，现在不要管什么能不能过试用期了，也不要管自己能不能适应公司的节奏了，什么也别管了，就牢牢记住一个原则：**”不懂就问！！！”，千万别不好意思！！！！自己不爱主动和别人说话这个毛病一定要改改了。因为你是新手问的话，他们不会太在意，等到时候你做了一段时间后再去问别人，他们就会怀疑你的能力了。**)

(注4：不要关注结果，关注过程！！！真实项目带给我的汲养绝对是巨大的，那么复杂的一个项目，一个新手想想都不可能完全理清其中的逻辑的。但是就在那一天天和真实项目接触的过程中，只要你勤学好问，不知不觉你就会成长了，所以能不能过试用期什么的结果，和自己能不能取得进步这一过程相比，这个根本就不值得一提了。)

(注5：刚刚我奶奶给我打电话，说我要是没有找到工作可以先回家一趟看看吗？我有种不好的预感。我过年也没有回家，现在刚找到工作，也一时半会回不去了。时间啊时间，总是过得这么快，我也接近不惑之年了，祖辈、父辈也渐渐老了，离死亡也更近了一步了。所以还是好好过接下来的每一天吧，珍惜时间吧。)

3.15(周一)

  • 本来今天是入职的，但是情况有变，HR(和同事聊天才知道，HR就是郭哥的老婆，兼任财务，每个月初4,5号给我们发工资)通知我明天入职，于是今天依旧待在家里。
  • 今天北京刮沙尘暴了，今天刚好没有上班，运气还挺好的。
  • 今天大概就是看了一下《JavaScript高级程序设计》，做了笔记。主要看了AJAX和Promise部分。

3.16(周二)

  • 今天正式上班，除我以外同事一共有7个(公司领导叫郭哥，今天他去见客户了，所以上午的时候并没有见到他)。和同事也简单聊了一下，发现我前面离职的那个前端也是安徽芜湖的，感觉冥冥之中有些巧合啊。
  • 刚开始第一天我主要是接触新环境，认识一下新同事。同事帮我办了一下门禁卡，告诉我工位在哪里，然后还建议我找郭哥要一下以前做过的项目，大概看一下。我微信找了，他让我自己先去找一下，我现在也就先只能大体走马光花的看一遍我这台电脑里的文件了。
  • 公司上班不打卡，属于弹性工作制，早上9:00上班，你只要不是上班时间太离谱，基本上都是可以的。晚上是下午6:00下班，我现在还没问中午休息多长时间。
  • 感觉上班氛围还是挺好的，虽然全部都是男生，不过氛围也没有很沉闷。我估计我应该比他们很多人都大，而且性格方便我感觉自己有点太腼腆了，虽然步入社会4年多了，人际交往方面还是一点也不主动，这个方面我需要慢慢调整一下。
  • 公司地址是在一个商铺里面，是商住两用的，第一次来面试的时候感觉空气有点闷，不过我这里右手边是有窗户的，是有新鲜空气的，所以还是可以的。公司一楼是办公的地方，楼上我之前以为是行政、财务工作的地方，结果不是，公司行政、财务就只有一个人，是郭哥老婆，而且好像还是在家办公。楼上主要就是堆放的一些杂物，估计洗手间也是在楼上吧。
  • 现在是11:10，我把这台电脑大体设置了一下，安装了一些我比较常用的软件，工作电脑屏幕很大，而且是双屏的，我一开始还不太适应。然后我就先把双屏给关了，现在就只用这一个大屏。
  • 同事告诉我说公司的主要用的技术是WebGL做一些展示效果，还给了我一本《Three.js开发指南》让我学习一下。我现在大体知道了，我现在的主要精力就是要快速上手Three.js了，今天晚上回家就开始学。
  • 我现在对同事还不算特别熟悉，只是大体了解了一下。
    • 同事中(其他同事说他应该算是郭哥的合作伙伴)好像有个新疆的，名字叫苏拉依·巴特尔，人也长得像新疆的。
    • 我左边的不知道叫什么名字，他是做三维模型的。
    • 我后边的负责平面设计，我以后应该会经常和他打交道的。今天早上他也是第一个来的，给我开了门。
    • 我后边的左边已经在这家公司干了4年了，是做Unity开发的的。
    • 我前面好像也是做Unity开发的，前面的左边还不太清楚。
    • 郭哥也说之后还会再来一个前端，我觉得到时候我们也有个照应，挺好的。不过自己也要快速适应一下公司和公司项目了，要尽早开始进行代码的产出了。
  • 我这个录屏感觉还是不要一直录，最好一个小时录一次，这样比较好。录屏的目的一方面是为了记录一下我的新的职业生涯，一方面也是留下一些痕迹，到时候我写代码的时候也录个屏，以后的自己可以看看现在的自己代码写的怎么样，有没有进步。我之前已经买了一个4T的超大移动硬盘，录一个小时算1个G左右的话，我能录好久好久，所以我无所畏惧。
  • 现在是中午11:38，上午时间基本上过完了，我的工作电脑也基本上设置的差不多了，接下来下午要做的，就是找一下这台电脑里面做过的项目了，看自己能不能把它给运行起来，看看效果。然后，大体了解一下Three.js，赶快抓紧时间去学一下啊。这个就是我今天下午要做的事情了。
  • 我觉得我也很有必要大体去了解一下Unity是什么，方便以后进行交流。
  • 郭哥回来了，给我展示了一下我的工作电脑里面的项目。跟我说了一下我现在要学的东西和侧重点。现在主要学的是D2Admin，一个用Vue做的后台管理系统，还说现在学Three.js还为时过早了，然后给我打开了2个文件夹，让我重点看一下其中一个，另一个顺带看一下即可。
  • 我还要学一下vh，这个我看项目里面也有很多。

  • 今天上班打算先看一下文档，照着先敲一下，郭哥说了我们现在只用了里面静态页面的部分，其他的很多东西都没有用过，所以让我直接去看项目，那我也只能快速看项目了，感觉进度比我想像的更快啊，不能像以前在家那样慢悠悠的学习了。

  • 我现在主要就是学习d2quanjing这个项目了。使用vue-devtools快速过一遍,不过这个工具我还不太熟，用的不算多。

  • VSCode我来下一个写注释自动添加日期的插件，暂且选了koroFileHeader，先用用看。

  • VSCode还有个书签功能，还没有用过，以后试着用一下。

  • VS Code文件内容比较功能，支持同屏滚动

  • VSCode中有个TODO插件，很好用。在平时的使用过程中，先在需要标记的地方写上TODO:xxxxxx 记得一定是大写的TODO，并且是英文状态下的冒号。当写完之后，TODO:会以高亮的方式显示。非常方便用于以后的查找。

  • vscode 中TODO的使用

  • 郭哥用的是Lean Cloud，用来创建数据库什么的，这个东西我以前听说过，但是从来没用过。

  • 今天认识到一个东西叫MetaHuman，郭哥说这个是未来的方向，突然感觉WebGL学好了真的很有用啊。

  • MetaHuman Creator前瞻：轻松创作高保真数字人类

  • MetaHuman Creator，让每个人都有一双“上帝造人之手”

  • MetaHuman官网

  • 我下载了一下这个软件，发现自己电脑配置显卡不行实在是卡，这个就只能等自己以后换一个好一点的电脑后再来试一下了。

项目相关

  • 我先找郭哥大体了解了一下项目需求，郭哥只跟我说了一点，就是那个展览管理应该在上面，点击它的话会弹出一个选框，下面的展厅管理，内容管理都应该在展览管理的基础之上进行。就是那个我不然还是先熟悉一下D2的基本用法吧，不然的话就算项目需求知道了也不会弄啊。

  • menu文件夹我看了一下实际项目和模版有些不一样，aside.js和header.js都没了，只剩下一个index.js，简化了很多,不知道是不是和下面的有关。

  • 这个项目应该是用的简化版的D2Admin，而且里面也就用了它很少的一部分功能，比如路由啊，页面啊什么的。

  • 郭哥让我先不用看文档了，直接看项目，还说东西是学不完的，遇到问题了再去查资料，看文档。但是我现在面对这个项目还是无从下手啊，起码必须要先稍微看一下文档看一下大体这个项目是如何做的吧。

  • 问了一下郭哥，这边项目都是有时间限制的，基本上都是控制在一个月以内。不过我这个项目暂时还没有时间限制。我现在还需要知道更多的信息，才能继续开展我的这个项目。比如客户的需求文档、最终要达到的效果图之类的。郭哥说了他到时候还要设计一下数据库，写一个接口地址，然后给我去用。到时候还存在如何使用接口的问题。

  • 这个项目从开展到现在，不知道已经过去多少天了，也不知道我现在又还剩下多少时间去做这个项目呢。

  • 项目的icon图标库在哪里？如果里面没有我想要的图标，我可以从外部引入吗？对了，它这个项目的图标好像就是引用的font-awesome，icon的可选至是font-awesome的所有图标名。

  • 项目首页的验证码每次登陆都自动变化，已经和原版不一样了。项目首页的验证码是在哪里设置的？

  • 项目的路由如下：

    • index：主页
    • editpanos：展厅管理
    • picedit：展版管理
    • videoedit：视频管理
    • iconedit：热点管理
    • panosedit：平面图
    • exhibitionlist：展厅列表
    • accountmanage:账户管理

  • 因为我现在要做的这个项目今天郭哥给我演示的时候，突然发现用户权限方面有点问题，后台需要改一下，就说这个项目先暂停，他到时候再把后台给弄一个，让我再看另一个项目去了。

领导布置的任务

  • 前面的D2Admin后台管理系统因为权限问题郭哥说先暂停吧，他先把后台解决一下再说，现在交给我的任务是给我们公司的官网做一个优化。
  • 这个官网是个可视化的展示效果，领导让我对比一个他之前给别人做的那个可视化项目进行参考。
  • 当前我们公司官网需要进行的优化有：
    • 必须等资源全部加载完后才显示进入按钮。(现在是进入按钮一打开页面就有，而页面资源却是加载了25秒资源才全部加载完，资源没加载完之前你点击按钮页面是没有反应的，这样会让别人以为页面卡了，然后退出了。)
    • 墙上的图片点击不了，要优化成要可以点(超链接)，要显示出图片和视频(航天信息这个项目做了，可以参考一下这个项目)
    • 第三件事，能不能把页面封装成一个APP或者小程序(做完前面2件事在做这个)，到时候给别人发一个网址或者App压缩包别人就可以在手机上打开了。

3.18(周四)

  • 我现在遇到了一个问题，Git提交仓库的代码太大，超过4G，然后它就提示remote: fatal: pack exceeds maximum allowed size，使用SVN就更不行了，它只有100MB的免费空间，然后我就想到了VSCode中有没有什么插件可以用来公司、家里电脑同步开发代码。稍微搜了一下，发现确实有相关的插件，叫VS Code Remote，不过现在是凌晨1:35了，之前睡了2个小时，所以今天熬了一会夜，现在时间也不早了，今天就这样吧，明天再弄了。
  • 这才是真正的远程开发——VS Code Remote 环境搭建
  • 郭哥说商业公司一般不用这个，因为安全性的考虑，到时候要是远程的话就开向日葵或Teamviewer。
  • 郭哥让我抓紧一下速度，我现在是不能再这么慢吞吞的了，交待的任务要感觉去做了。
  • Typora添加删除线快捷键是Alt+Shift+5，以后问题解决了就添加删除线。
  • 公司还有一个静态官网，地址为http://banlantech.icoc.me/，也是用Vue做的，我有时间也学一下这个官网是怎么写的。
  • 郭哥跟我说了，一切事情都要分个轻重缓急，优先保证功能完成。不要有过多的代码洁癖，不要钻牛角尖。——我跟郭哥说有一些无效CSS，要把它删掉时郭哥跟我说的。那我就先把功能给完成吧，这些事情就以后再说了。

3.19(周五)

  • 郭哥昨天跟我说了我们这里主要是敏捷开发，也就是说只要完成功能就好了，其他的什么性能优化什么、代码洁癖什么的都不重要。我们现在最重要的就是完成一个又一个功能就好了。
  • 我现在已经是在工作了，所以思想要进行转变了。我不能面对一个新项目从头到尾系统的学习了，不能对着官方API文档一个一个看了。现在只能是直接就上手干了，等遇到问题了才能去看官方文档或者请教别人等各种方式来解决这个问题。
  • 那我现在就抱着一个宗旨，我所做的一切都是以解决布置给我任务中的一个一个的问题最终完成整个项目为原则的，其他的和我当前任务无关的部分一概都不管。这个就是我的第一个准则了吧。
  • 今天，跟智明要了一下官网所需要的资料。我先大体看一下资料里面的图片和视频吧。

3.20(周六)

  • 今天睡了个饱，现在是下午2点52，洗了个澡，现在身体清清爽爽，刚刚点了个外卖，还在送的途中。今天接下来的时间我就要开始再学习一下Vue及做有关项目了。着重学习一下vue-awesome-swiper这个轮播图组件，自己的水平还很不够啊，还需要多多敲代码啊。
